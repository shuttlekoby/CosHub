# Twitter メディア ダウンローダー (twmd)

![](https://img.shields.io/badge/version-1.14.2-blue)
![](https://img.shields.io/badge/platform-macOS%20%7C%20Windows%20%7C%20Linux-green)
![](https://img.shields.io/badge/language-Go-00ADD8)

**APIキーなしでTwitterの画像・動画を簡単ダウンロード！**

このツールは、Twitter APIキーを必要とせずに、Twitterの画像や動画を効率的にダウンロードできるコマンドラインツールです。

## ✨ 主な機能

- 🚀 **APIキー不要** - Twitterの認証情報なしで使用可能
- 📸 **画像ダウンロード** - 高画質な画像を一括取得
- 🎥 **動画ダウンロード** - MP4形式の動画をダウンロード
- 👤 **ユーザー単位** - 特定ユーザーの全メディアを取得
- 🐦 **単一ツイート** - 個別ツイートからメディアを抽出
- 🔄 **リツイート対応** - リツイートも含めて取得可能
- 📁 **カスタム保存** - ファイル名・保存先を自由に設定
- 🔒 **NSFW対応** - ログインしてセンシティブコンテンツも取得

## 🛠 インストール方法

### 前提条件
- macOS 10.12以降
- インターネット接続

### 自動セットアップ（推奨）

すでにビルド済みの`twmd`バイナリが利用可能です：

```bash
# 実行権限を確認
chmod +x twmd

# ヘルプを表示して動作確認
./twmd -h
```

### 手動ビルド（上級者向け）

Go言語がインストールされている場合：

```bash
# リポジトリをクローン
git clone https://github.com/mmpx12/twitter-media-downloader.git
cd twitter-media-downloader

# ビルド
make

# インストール（オプション）
sudo make install
```

## 🚀 基本的な使い方

### 1. 特定ユーザーの画像・動画を全てダウンロード

```bash
./twmd -u ユーザー名 -o ~/Downloads -a -n 300
```

**解説：**
- `-u ユーザー名`: ダウンロード対象のTwitterユーザー名
- `-o ~/Downloads`: 保存先フォルダ
- `-a`: 画像と動画の両方をダウンロード
- `-n 300`: 最大300ツイートまで取得

### 2. 単一ツイートからメディアをダウンロード

```bash
./twmd -t 1234567890123456789
```

**解説：**
- `-t`: ツイートIDを指定
- ツイートIDはTwitterのURL末尾の数字部分

### 3. 画像のみダウンロード

```bash
./twmd -u ユーザー名 -i -n 100 -o ./images
```

### 4. 動画のみダウンロード

```bash
./twmd -u ユーザー名 -v -n 50 -o ./videos
```

## ⚙️ 詳細オプション

### 基本オプション

| オプション | 説明 | 例 |
|------------|------|-----|
| `-h, --help` | ヘルプを表示 | `./twmd -h` |
| `-u, --user` | ユーザー名を指定 | `-u elonmusk` |
| `-t, --tweet` | ツイートIDを指定 | `-t 1234567890` |
| `-n, --nbr` | ダウンロード数の上限 | `-n 500` |
| `-o, --output` | 保存先ディレクトリ | `-o ~/Downloads` |

### メディア種別

| オプション | 説明 |
|------------|------|
| `-i, --img` | 画像のみダウンロード |
| `-v, --video` | 動画のみダウンロード |
| `-a, --all` | 画像と動画の両方 |

### 追加機能

| オプション | 説明 | 例 |
|------------|------|-----|
| `-r, --retweet` | リツイートも含める | `-r` |
| `-R, --retweet-only` | リツイートのみ | `-R` |
| `-M, --mediatweet-only` | メディア付きツイートのみ | `-M` |
| `-U, --update` | 未ダウンロードのメディアのみ | `-U` |
| `-z, --url` | URLのみ表示（ダウンロードしない） | `-z` |

### ファイル設定

| オプション | 説明 | 例 |
|------------|------|-----|
| `-s, --size` | 画像サイズ（small/normal/large） | `-s large` |
| `-f, --file-format` | ファイル名形式 | `-f "{DATE}_{USERNAME}_{ID}"` |
| `-d, --date-format` | 日付形式 | `-d "2006-01-02_15-04-05"` |

### 認証・ネットワーク

| オプション | 説明 | 例 |
|------------|------|-----|
| `-L, --login` | ログイン（NSFW用） | `-L` |
| `-C, --cookies` | Cookie認証 | `-C` |
| `-p, --proxy` | プロキシ使用 | `-p socks5://127.0.0.1:9050` |

## 📝 実用的な使用例

### 例1：お気に入りのアーティストの作品を収集

```bash
# 画像のみ、最新500ツイートから
./twmd -u artist_name -i -n 500 -o ~/Pictures/Artists -s large
```

### 例2：ニュースアカウントの動画をアーカイブ

```bash
# 動画のみ、リツイート含む
./twmd -u news_account -v -r -n 200 -o ~/Videos/News
```

### 例3：特定のツイートスレッドを保存

```bash
# 個別ツイート、ファイル名にユーザー名と日付を含める
./twmd -t 1234567890 -f "{USERNAME}_{DATE}_{ID}" -d "2006-01-02"
```

### 例4：定期的な更新ダウンロード

```bash
# 新しいメディアのみ取得（差分更新）
./twmd -u favorite_user -a -U -n 1000 -o ~/Twitter_Archive
```

## 🔐 プライベート・NSFWコンテンツの取得

### Cookie認証の設定方法

1. **Twitterにブラウザでログイン**
2. **開発者ツールを開く** (F12キー)
3. **Storageタブ → Cookies → twitter.com**
4. **以下の値をコピー:**
   - `auth_token`
   - `ct0`

5. **認証ファイルを作成:**

```bash
# twmd_cookies.jsonファイルを作成
echo '{
  "auth_token": "あなたのauth_token",
  "ct0": "あなたのct0"
}' > twmd_cookies.json
```

6. **Cookie認証でダウンロード:**

```bash
./twmd -C -u private_user -a -n 100
```

## ⚠️ 重要な注意事項

### レート制限について

- **最大取得数**: 500-600ツイート程度（Twitterの制限）
- **推奨間隔**: 大量ダウンロード時は適度な間隔を空ける
- **プロキシ使用**: 必要に応じてプロキシ経由でアクセス

### 法的・倫理的な考慮事項

- ✅ **個人利用**: 問題なし
- ✅ **研究目的**: 適切な範囲で利用
- ❌ **商用利用**: 著作権に注意
- ❌ **大量取得**: サーバー負荷を考慮
- ❌ **再配布**: 他者の著作物の無断配布は禁止

### ファイル保存について

- デフォルトで現在のディレクトリに保存
- `-o`オプションで保存先を指定推奨
- 同名ファイルは上書きされるので注意

## 🐛 トラブルシューティング

### よくある問題と解決法

#### 1. "command not found" エラー

```bash
# 実行権限を確認
chmod +x twmd

# パス指定で実行
./twmd -h
```

#### 2. ダウンロードできない

```bash
# ユーザー名を正確に入力
./twmd -u correct_username -a -n 10

# ツイートIDを確認
./twmd -t 正確なツイートID
```

#### 3. プライベートアカウントにアクセスできない

```bash
# Cookie認証を使用
./twmd -C -u private_account -a -n 10
```

#### 4. 動画ダウンロードが失敗する

```bash
# 画像のみで試す
./twmd -u username -i -n 10

# 個別ツイートで試す
./twmd -t tweet_id
```

### デバッグモード

詳細なエラー情報が必要な場合：

```bash
# バナー非表示でエラーを確認
./twmd -B -u username -a -n 5
```

## 📊 パフォーマンス最適化

### 効率的なダウンロードのコツ

1. **適切な数量指定**: `-n 100-500`程度が推奨
2. **メディア種別を絞る**: `-i`または`-v`で対象を限定
3. **更新モード活用**: `-U`で差分ダウンロード
4. **ファイル名最適化**: `-f`でわかりやすい命名
